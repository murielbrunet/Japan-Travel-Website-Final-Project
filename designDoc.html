Major Design Goals

My main goal in creating this website was to create a space in which all the different resources on Japan could be collected. Japan is very centered around seasons in a variety of ways so I thought it would make the most sense with the theme to create different skins, each one corresponding to a season. There is an entry page before the homepage of the website to ask the user to pick a season. The home page showcases a variety of photographs, which would eventually link to resources on the resource page. The resource page is made to show a variety of different links and useful information. Right now, it showcases two Japanese life vloggers, with a popular video imbedded into the website and a link to their YouTube channel. The Events page holds a calendar to allow users to get to know which events are coming up. Finally, the Contact page holds a form to allow users to sign up for a newsletter. Overall, the website is rather straightforward as it is made up of four pages, all laid out in the same way. What makes this website somewhat different from others is its use of images and colors to create the look and feel of the four different seasons of Japan.

I made use a variety of resources to complete this project. I edited my code on Sublime Text 2. Furthermore, I use W3Schools a lot for reference on how to create links, use buttons, have javascript events, and the like. I also made great use of Color Hex, a website that shows a huge variety of colors and their hex numbers, which made it easier for me to create my themes. Finally, I also made use of Google Help to find out how to imbed YouTube and Google Calendar into my website.

I tried to use percentages as much as possible in order to accommodate for varied browser sizes. Lastly, this website works on both Safari and Chrome, though I was unable to test it on Firefox as I do not have ready access to that browser.

Storage

I used session storage to keep track of the theme chosen by the user. In this way, the user could choose a theme upon entering the website and then have the theme remain the same throughout their use of the site. They also had the option of changing the theme by clicking on a navbar at the bottom of the page. I used session storage so that the user would have to choose a new theme every time they visited the website. The default theme, in case the user doesn't enter the site through the intro page, is set to spring.

Interaction

There is a variety of interaction throughout the website. The navbar options change colors once the user scrolls over the options to let the user know that these are clickable. This is especially useful for the theme navbar at the bottom of the page because the themes are not links, so the hover is all the more important in letting the user know that they are clickable any way. The themes are actually just a set of four different css pages that are changed depending on the value stored under 'season' in the session storage. The appropriate css file is chosen at the load time of each page. There is also added interactivity through clickable images in the resources page as well as a calendar that people can interact with the understand the variety of events coming up in the near future. Furthermore, the example information within the form disappears when the user clicks it to fill it out. This was done use javascript and examples from class.

User Feedback

The main user feedback employed is the theme. By getting the user to choose a theme, they can customize their experience of the website. There is also some user feedback with the submission of the form, which allows the user to see that they submitted the correct data before returning the the home page.

Multimedia

There is a variety of multimedia used. I used photos as the background of the website as well as embedded some YouTube videos to show examples of Japanese vloggers. I also added an embedded Google calendar to display various events happening in relation to Japan that might be of interest to the user.
